The Command Line has several different modes of execution

Mode 1\+: The most standard mode requires that a partition name be passed as an argument. The partition name must exist on the filesystem if it does not, the commandline will quit. This mode expects the user to manually type commands into the command line interface. This mode\textquotesingle{}s run command looks like\+: \textquotesingle{}./commandline Partition\+Name\textquotesingle{}

Mode 2\+: The second mode adds debugging information to Mode 1 The flag that must be passed to enable this mode is \textquotesingle{}-\/d\textquotesingle{} This mode\textquotesingle{}s run command looks like\+: \textquotesingle{}./commandline Partition\+Name -\/d\textquotesingle{}

Mode 3\+: The third mode operates simmilar to Mode 1 except that rather than expecting users to manually type commands in, it expects a file containing all of the commands that will be executed, to be piped to it. This mode still requires that a legal partition be passed. The flag that must be passed to enable this mode is \textquotesingle{}-\/d\textquotesingle{} This mode\textquotesingle{}s run command looks like\+: \textquotesingle{}.commandline Partition\+Name -\/s\textquotesingle{}

Mode 4\+: The fourth and final mode adds debugging support to Mode 3. The flag that must be passed to enable this mode is \textquotesingle{}-\/d\textquotesingle{} This mode\textquotesingle{}s run command looks like\+: \textquotesingle{}./commandline Partition\+Name -\/s -\/d\textquotesingle{}
